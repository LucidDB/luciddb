Farrago

Farrago is an attempt to build an enterprise-class open-source RDBMS with a modern architecture. Why does the world need another open-source RDBMS when there are so many already? Please see the Project FAQ. A major goal is to do as little work as possible. Sometimes being lazy is good, and in this case many of the required components are already available as open-source, and just waiting to be put together. For the parts that need custom coding, some of the ideas from model-driven architecture can save a lot of gruntwork. Currently, just a skeleton of the full system exists. The project roadmap defines how to get to a fully functional system.

The project is hosted at SourceForge.

The source code is hosted in John Sichi's perforce server. A big thanks to Perforce for letting us use p4d gratis.

Contributing

No matter what background you're coming from, Farrago Wants You!
  • Do you like bits, bytes, bus errors, and memory barriers? There's some code waiting for you somewhere in Fennel (if you're a hardcore kernel hacker, aren't you tired of reinventing the linked list yet?).
  • Application developer? Creating a table isn't much different from creating a new customer account, and the design patterns employed in Farrago and Fennel should make you feel right at home. Or if you're into generic Java server technology, work on integrating Farrago into your favorite app server.
  • Client side? Work on JDBC or ODBC drivers, or make Farrago show up in one of the many management platforms out there.
  • Script-junkie? You name it, we probably need it.
  • Do the prison cell aesthetics of this website offend you? That's intentional. Help do something about it!
  • ...
If you're interested, read through the getting started docs and send mail to perfecthash-AT-users.sf.net to get brainwashed into contributing.
